# מדריך תפעול קצר

## הוספת פודקאסט חדש או מקור נוסף

1. פותחים את טופס ההצטרפות:

   ```text
   https://shaqo88.github.io/youtube-podcast-feeds/onboard/
   ```

2. בוחרים איפה נמצאים השיעורים:

   - יוטיוב
   - תיקיית Google Drive
   - פיד פודקאסט קיים
   - שילוב של כמה מקורות לאותו פודקאסט

   בשדה יוטיוב אפשר להדביק ערוץ או פלייליסט. אם הקישור הוא פלייליסט,
   המערכת תזהה זאת לבד.

3. ממלאים את הפרטים. השדה `שם קצר לקישור באנגלית` חובה, לדוגמה:

   ```text
   rav-example
   ```

4. אם זה פודקאסט חדש, משתמשים בשם קצר חדש.
5. אם רוצים להוסיף מקור לפודקאסט קיים, משתמשים באותו שם קצר שכבר קיים.
   אחרי אישור, המערכת תוסיף רק את המקור החדש ולא תיצור פודקאסט נוסף.
6. אחרי השליחה נוצר Issue בגיטהאב עם התווית `needs-approval`.

## אישור פודקאסט מתיקיית Drive

1. מריצים:

   ```text
   Actions -> Check Drive Folder -> Run workflow
   ```

2. מדביקים את קישור התיקייה ובודקים שיש לפחות קובץ אחד לפרסום.
3. אם הכל תקין, מוסיפים ל-Issue את התווית:

   ```text
   approved
   ```

4. GitHub Actions יוצר את הפודקאסט, מסנכרן פרקים ראשונים, מפרסם Feed,
   מוסיף תגובה עם הקישור, מסיר `needs-approval`, וסוגר את ה-Issue.

## אישור פודקאסט מיוטיוב

1. פותחים את קישור היוטיוב ובודקים שזה הערוץ או הפלייליסט הנכון.
2. אם הכל תקין, מוסיפים ל-Issue את התווית:

   ```text
   approved
   ```

3. המערכת תנסה לזהות את מזהה הערוץ או הפלייליסט, לקחת תמונה מיוטיוב אם לא
   סופקה תמונה, ליצור או לעדכן Config, לסנכרן פרקים ראשונים, ולפרסם Feed.

## אישור פודקאסט משולב יוטיוב + Drive

1. בודקים את מקור היוטיוב.
2. מריצים `Check Drive Folder` על תיקיית ה-Drive.
3. אם שניהם תקינים, מוסיפים ל-Issue את התווית `approved`.
4. שני המקורות יסונכרנו לאותו Feed.

## אישור פודקאסט מפיד קיים

1. פותחים את קישור הפיד ובודקים שהוא RSS או Atom תקין עם קבצי אודיו.
2. אם זה מקור נוסף לפודקאסט קיים, משתמשים באותו שם קצר באנגלית.
3. אם הכל תקין, מוסיפים ל-Issue את התווית:

   ```text
   approved
   ```

4. המערכת תוסיף מקור מסוג `existing_feed`, תעתיק את קבצי האודיו ל-R2,
   ותפרסם אותם בפיד של Torah Pod.

## הוספת מקור לפודקאסט קיים

1. בטופס משתמשים באותו `שם קצר לקישור באנגלית` של הפודקאסט הקיים.
2. בוחרים את המקור החדש או שילוב המקורות.
3. אחרי אישור, המערכת מוסיפה ל-Config רק מקורות שלא קיימים עדיין.
4. הפרקים החדשים והישנים מופיעים יחד באותו RSS, ממוינים לפי תאריך.

## מחיקת פודקאסט ניסיון

1. מוחקים את התיקיות:

   ```text
   shows/<slug>
   public/<slug>
   ```

2. דוחפים את השינוי ל-`main`.
3. מוחקים את קבצי האודיו מ-R2:

   ```text
   Actions -> Delete R2 Prefix -> Run workflow
   ```

4. מכניסים:

   ```text
   prefix=<slug>
   confirm=DELETE
   ```

## Nachmanson

הפודקאסט של הרב נחמנסון הועתק לריפו הזה עם הפרקים והתמונה מה-ripo הישן.
כרגע הוא מוגדר כ-`enabled: false`, כי מזהה הפלייליסט היה שמור כ-GitHub Secret
בריפו הישן ואי אפשר לקרוא ערך של Secret קיים. כדי להפעיל סנכרון עתידי צריך
להכניס את מזהה הפלייליסט ל-`shows/nachmanson/config.yml` ולהחליף
`enabled: false` ל-`enabled: true`.

לפני מעבר מלא מומלץ להעביר את קבצי האודיו ל-R2 הסטנדרטי:

```text
Actions -> Migrate Existing Media To R2 Prefix -> Run workflow
show=nachmanson
dry_run=true
```

אם ה-Dry run נראה תקין, מריצים שוב עם:

```text
dry_run=false
```
